# Security policy

`parser-go` is maintained in public at `github.com/sagaragas/parser-go`. The project started as a mission-mode clean-room rewrite, so please report issues against this repo's code and artifacts, and keep private data out of public threads.

## Supported line

Until tagged releases begin, security fixes land on `main`. Please reproduce against the latest `main` commit before you open a report when you can.

## Reporting a vulnerability

Use GitHub's private vulnerability reporting flow for `github.com/sagaragas/parser-go` when it is available. If you cannot open a private report, do not post exploit details, credentials, or raw logs in a public issue. Open a short public issue that asks for a private follow-up path instead.

## What to include

Please include:

- the affected commit, tag, or reproduction revision
- the impact you observed
- a minimal reproduction or request path
- any logs or payloads in redacted form

Please remove tokens, cookies, authorization headers, internal-only URLs, and raw homelab log data before you share anything.

## Response target

We aim to acknowledge reports within 3 business days and will share status updates until we have a fix, a mitigation, or a clear reason why the report does not apply.
